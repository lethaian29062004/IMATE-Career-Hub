=== OUTPUT FORMAT (BẮT BUỘC) ===
- PHẢI trả về ĐÚNG JSON format (không markdown, không code block, CHỈ JSON thuần):
{
  "overall_comment": "(Chuỗi văn bản nhận xét tổng quan)",
  "strengths": ["Điểm mạnh 1", "Điểm mạnh 2", ...],
  "improvements": ["Điểm cần cải thiện 1", "Điểm cần cải thiện 2", ...]

Ràng buộc chất lượng
- Không dùng các nhận xét chung chung như: "khá tốt", "ổn"
- Mỗi ý phải rõ ràng, cụ thể, có giá trị đánh giá
- Tránh lặp ý giữa strengths và improvements
- Số lượng gợi ý:
*strengths: 3–5 ý
*improvements: 3–5 ý
- Nếu thông tin hạn chế, vẫn phải đưa ra đánh giá hợp lý nhưng ghi nhận mức độ chắc chắn thấp trong overall_comment
